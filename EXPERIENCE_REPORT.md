## Task 23 新增修复（白狼王技能+发言历史+事件日志+UI重构）

### 新问题1: 白狼王白天技能按钮缺失
**根因**: 白狼王自爆按钮仅在 `userAction === 'speak'`（自己发言轮）时显示，无法在白天任意时刻主动发动。

**修复**:
- 顶部 header 新增**常驻自爆按钮**: `user.role==='white-wolf' && whiteWolfSkillAvailable && (phase==='day-discuss' || phase==='day-vote')`
- `userSelfDestruct` 重构为双入口兼容: 系统触发(AI) + 用户主动(检测角色&技能可用)
- `userAction` 逻辑调整: `selfDestructMode` 优先于 `speak`，发言轮也可切自爆
- AI 白狼王主动自爆: `day>=2` 启发式评估（难度分档概率 8%/12%/18%，劣势×1.5，优先打暴露神职）

### 新问题2: 发言历史过夜丢失
**根因**: 每次 `speeches: []` 清空直接丢弃，无归档机制。

**修复**:
- 新增 `SpeechRound` 类型 + `speechHistory[]` 字段
- `_archiveSpeeches(label)` helper: 仅在非空时归档，14 个清空点全覆盖
- 归档标签: "第X天·竞选警长" / "第X天·白天讨论" / "第X天·遗言"
- InfoPanel 讨论 Tab: 当前轮 + 历史轮次（Collapsible 可折叠，带数量徽章）
- 讨论区快捷入口: "历史N轮" 按钮 + 空讨论时历史提示卡片

### 新问题3: 事件记录不可见
**根因**: 关键事件散落在 log 中，无独立事件流。

**修复**:
- 新增 `GameEvent` 类型 + `events[]` 字段
- 6 类事件全程记录:
  | category | 场景 | icon |
  |----------|------|------|
  | phase | 夜降临/天亮/平安夜 | 🌙☀️ |
  | death | 夜间死亡/被带走/被射杀 | 💀 |
  | sheriff | 当选/无人上警/平票/移交/撕毁 | 🏛️👑 |
  | vote | 被放逐/平票/弃票 | 🗳️ |
  | skill | 白狼王自爆/猎人开枪 | 💥🏹 |
  | result | 胜负判定 | 🎉 |
- InfoPanel 事件 Tab: 倒序展示（最新在上），按 category 着色边框

### UI 重构: 三合一信息面板
**原**: 单一日志 Sheet（仅 GameLog）
**新**: 多 Tab 信息面板 `InfoPanel.tsx`
- Tab "💬 讨论": 当前轮发言 + 历史轮次折叠列表
- Tab "🔔 事件": 事件流（倒序、分类着色、含详情）
- Tab "📋 日志": 完整游戏日志（保留原 GameLog）
- Tab 数量徽章 + 信息按钮红点提示（有新事件/历史时）

## 一、本轮修复总结（Task 22）

### 问题1: 发言内容水、不全
| 维度 | 修复前 | 修复后 |
|------|--------|--------|
| 发言长度 | 30-80字 | 60-150字 |
| 角色策略 | 1行通用描述 | 每角色×3档难度=18种策略 |
| 发言重复 | 严重复读(后3人几乎相同) | 发言位置感知+个性化风格+严禁复读 |
| 上下文 | 仅引用发言记录 | 识别预言家对跳、警徽流、死亡分析 |
| 显示区域 | h-40(160px) | h-56(224px)+自动滚底 |

**关键改进**:
- 发言位置感知: 第1人定调、中间人必须带新角度(同意补充/反驳/新疑点)、最后人总结
- 4种个性化风格(逻辑缜密/直觉敏锐/激进冲锋/稳健保守)按玩家ID分配
- 预言家强制报查验结果+警徽流(验X撕Y格式)
- 前序发言关键观点提取+严禁重复指令

### 问题2: 没人上警
**根因**: `_aiSheriffDecided`标志被误用——`userJoinSheriff`立即设true,导致`proceedDayPhase`的AI收集分支被跳过,AI永远不上警。

**修复**:
- 新增`_aiSheriffCollected`标志,与用户决定标志分离
- 预言家上警概率0.9→0.95
- 保底机制: 若无AI上警且用户未上警,强制预言家(或随机AI)上警

### 问题3: 投票慢
**修复**: 30秒超时+倒计时UI+自动弃票
- `voteDeadline`时间戳 + 31秒setTimeout
- 倒计时组件(≤10秒红色脉冲)
- 超时自动调用`userVote(null)`弃票

### 问题4: AI策略档位
**3档策略系统**(保留easy/normal/hard字段,重新定义为玩法风格):

| 档位 | 风格 | 狼人策略 | 好人策略 |
|------|------|----------|----------|
| easy | 激进型 | 悍跳神职、强势带节奏、直接攻击 | 强硬站边、穷追猛打、敢于归票 |
| normal | 均衡型 | 1悍跳+冲锋/倒钩分配、制造合理怀疑 | 逻辑分析、跟随金水/查杀归票 |
| hard | 老练型 | 深水/倒钩、站边真预言家、制造平安夜 | 盘狼坑位、票型反推、轮次计算 |

### 问题5: 预言家卡死(上轮修复)
`confirmSeerResult`阶段设置错误(night-witch→night-end)

## 二、当前趣味性评估

### ✅ 做得好的部分
1. **完整游戏循环**: 夜→警长竞选→讨论→投票→夜...流畅运行,多日循环不卡死
2. **角色全覆盖**: 预言家/女巫/猎人/守卫/白狼王/平民,技能完整
3. **视角隔离**: AI决策输入不含其他玩家role,防作弊
4. **AI韧性**: 429重试+并发控制,限流时fallback决策仍可博弈
5. **昼夜过渡动画**: "天黑请闭眼"/"天亮了"仪式感强
6. **上下文AI状态**: "🐺狼人正在商议击杀目标..."比"处理中"更直观

### ⚠️ 待优化部分

#### P0 — 核心体验
1. **429限流致发言退化**: 6人局一天讨论需5次LLM调用,高峰时3/11退化为短fallback
   - 影响: 部分AI发言变水,破坏沉浸感
   - 建议: MAX_CONCURRENT=2→1; 或预生成发言批量调用

2. **AI幻觉**: 偶尔出现"我站X号预言家"但X号从未跳预言家
   - 影响: 逻辑出戏,降低可信度
   - 建议: prompt中明确列出"已跳预言家的玩家:无"而非省略

#### P1 — 策略深度
3. **警徽流执行**: 预言家死亡后警徽流应自动执行(撕警徽/移交),目前仅文字描述
   - 建议: 预言家死亡时检查nightAction.seerTarget,自动触发警徽流逻辑

4. **狼人战术单一**: 当前狼人主要"悍跳预言家",缺乏"深水狼"(装好人)、"倒钩狼"(站边真预言家)的明显区分
   - 建议: 困难局给狼人分配具体战术角色(悍跳/冲锋/倒钩/深水)

5. **投票逻辑浅**: AI投票主要看发言,缺乏票型分析、轮次计算
   - 建议: 困难局AI投票时传入历史票型,支持"改票""分票"战术

#### P2 — 体验细节
6. **发言节奏**: AI发言间隔800ms固定,缺乏节奏变化(紧张时快、关键发言慢)
7. **音效缺失**: 发言/投票/查验无音效反馈(仅有夜晚/死亡/胜利音效)
8. **无回看**: 死亡后只能看日志,缺乏"上帝视角"观察AI博弈
9. **平衡性**: 6人局(2狼+预+女+2民)狼人胜率偏高,建议调整或增加9人局推荐

## 三、优化开发路线图

### 第一优先级: 发言质量保障 (1-2天)
- [ ] LLM调用降并发: MAX_CONCURRENT 2→1,牺牲速度换质量
- [ ] 发言预生成: 白天开始时批量生成所有AI发言草稿,减少实时调用
- [ ] Fallback增强: 限流时根据角色+游戏状态生成有意义的占位发言(非通用模板)
- [ ] AI幻觉修复: prompt中显式声明"无人跳预言家"而非省略

### 第二优先级: 策略深度 (3-5天)
- [ ] 警徽流自动执行: 预言家死亡→检查seerTarget→撕警徽/移交
- [ ] 狼人战术分配: 困难局开局分配悍跳/冲锋/倒钩/深水角色
- [ ] 票型分析: 困难局AI投票时传入历史票型数据
- [ ] 女巫药剂策略: 基于预言家存活状态决定救药使用时机

### 第三优先级: 体验打磨 (持续)
- [ ] 发言节奏: 关键发言(跳预言家/对跳)后增加停顿
- [ ] 音效系统: 查验/投票/药剂使用音效
- [ ] 死亡观战: 死亡后可切换"上帝视角"看全部角色
- [ ] 平衡性调整: 6人局增加1民或改为1狼,降低狼人胜率
- [ ] 成就系统: "悍跳成功""盲毒狼人""盲守成功"等成就
